我們將進行導航重構，將「公共區域」整合進主導航欄並重命名為「評論」，同時優化評論區域的子導航。

### 1. 修改 `components/TabNav.tsx`
- **重構導航項目**：
    - 「歌」保持不變（對應個人區域 `/albums`）。
    - 新增「評論」（對應公共區域 `/albums?view=public`），與「歌」、「诗的歌」、「说明」並列。
- **高亮邏輯更新**：
    - 當路徑為 `/albums` 且無 `view` 參數時，高亮「歌」。
    - 當路徑為 `/albums` 且 `view=public` 時，高亮「評論」。

### 2. 修改 `app/albums/page.tsx`
- **移除舊切換按鈕**：刪除頁面頂部的「個人區域 / 公共區域」按鈕。
- **適配 URL 參數**：
    - 移除 `viewMode` state，改為從 `searchParams` 讀取 `view` 參數。
    - `view === 'public'` 時顯示公共區域（評論區）。
- **新增子菜單邏輯（僅在公共區域顯示）**：
    - 新增子導航： **Top 20** (默認) | **全部**。
    - **Top 20**：顯示原有的熱門專輯列表（帶圖片和評分）。
    - **全部**：顯示一個新的文字列表視圖（僅顯示專輯名/歌手），點擊後彈出詳情（圖片和評分）。

### 3. 修改 `components/AlbumGrid.tsx`
- **支援純文字列表模式**：
    - 新增 `layout` prop (`'grid' | 'list'`)。
    - 當 `layout='list'` 時，渲染緊湊的文字列表。
    - 點擊列表項時，同樣觸發 `handleSelectAlbum` 打開詳情彈窗。
- **數據源適配**：
    - 「全部」視圖需要調用新的 API（或重用現有 API 但參數不同），獲取所有專輯的精簡信息。

### 4. 後端 API 調整（如有需要）
- 檢查 `/api/albums/public/top` 是否足夠支持「全部」視圖，或需新增 `/api/albums/public/all`（分頁獲取全量數據）。為了性能，建議新增一個分頁接口。

這個方案將徹底清理頁面頂部的按鈕，使導航結構更加扁平化，符合您的需求。
