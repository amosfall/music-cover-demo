為了實現公共區域選出 Pick 最多的 20 張專輯並加入評論打分系統，我們需要對資料庫模型、後端 API 和前端介面進行一系列的升級。

### 1. 資料庫模型修改 (`prisma/schema.prisma`)
- **新增 `AlbumReview` 模型**：用於儲存用戶對專輯的評分和評論。
    - 欄位：`id` (主鍵), `albumName`, `artistName` (用於聚合不同用戶上傳的同一張專輯), `rating` (評分 1-5), `content` (評論內容), `userId` (評論者), `createdAt`。
    - **注意**：因為不同用戶可能上傳同一張專輯（會有不同的 `AlbumCover` 記錄），評論系統應該基於 `albumName` + `artistName` 進行聚合，而不是綁定到單個 `AlbumCover` ID，這樣才能算出該專輯在全站的總熱度。
- **更新資料庫**：執行 `prisma db push` 更新 Schema。

### 2. 後端 API 開發 (`app/api/albums/public/route.ts`)
- **新增聚合 API**：
    - 邏輯：統計全站 `AlbumCover` 表中出現次數最多的 `albumName` + `artistName` 組合。
    - 排序：依照「被收藏次數」（Pick 數）降序排列，取前 20 名。
    - 附加資訊：為每張熱門專輯附帶計算出的平均評分 (`averageRating`) 和評論數。

- **新增評論 API (`app/api/reviews/route.ts`)**：
    - `POST`：提交評分與評論。
    - `GET`：獲取特定專輯的評論列表。

### 3. 前端介面升級
- **公共區域改造 (`components/AlbumGrid.tsx` & `app/albums/page.tsx`)**：
    - 當 `scope="public"` 時，呼叫新的聚合 API，展示全站 Pick 最多的 20 張專輯。
    - 專輯卡片上顯示「被收藏次數」和「平均評分」。
- **評論互動 (`components/AlbumDetailModal.tsx`)**：
    - 點擊公共區域的專輯時，彈出的詳情頁中加入「評論區」和「打分組件」。
    - 顯示該專輯的總體評分和精選評論。

這個方案的核心在於**透過專輯名和歌手名來聚合資料**，將分散在每個人個人區域的專輯視為同一個「條目」，從而統計出全站最受歡迎的音樂。
