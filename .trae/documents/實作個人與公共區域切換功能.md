將「專輯」頁面改造成包含「個人區域」與「公共區域」的雙模式介面。

### 1. 後端 API 修改 (`app/api/albums/route.ts`)
- **新增 `scope` 參數支援**：
    - 接收 URL 查詢參數 `scope`。
    - 當 `scope=public` 時，**不依據 `userId` 過濾**，而是回傳所有使用者的最新專輯（限制數量，例如最新的 100 張），實現「公共區域」功能。
    - 當 `scope=personal`（預設）時，維持現有邏輯，只回傳當前登入使用者的專輯。

### 2. 組件修改 (`components/AlbumGrid.tsx`)
- **新增 `scope` 與 `readOnly` 屬性**：
    - `scope`：用於在呼叫 API 時傳遞 `?scope=public`。
    - `readOnly`：當為 `true` 時，**隱藏刪除按鈕**，防止使用者誤刪公共區域中他人的專輯。

### 3. 頁面邏輯修改 (`app/albums/page.tsx`)
- **新增檢視模式狀態**：加入 `viewMode` 狀態 (`'personal'` 或 `'public'`)。
- **新增切換按鈕**：在頁面頂部或工具列區域加入「個人區域 / 公共區域」切換開關。
- **介面連動調整**：
    - **個人模式**：顯示現有的分類標籤（Tab）、新增按鈕、清空按鈕、匯出功能。
    - **公共模式**：隱藏分類標籤（因為分類是個人的）、隱藏管理功能（新增、刪除、清空），僅展示 `AlbumGrid`（設為唯讀）。

這樣修改後，您就可以通過點擊按鈕在「只看自己」和「看所有人」之間切換，且公共區域的內容是唯讀的，確保資料安全。
